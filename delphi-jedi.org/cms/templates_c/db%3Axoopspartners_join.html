Project JEDI Alliance


Portal

Recently a number of people have approached Project JEDI, interested in having their projects become part of JEDI.

After discussion on the Steering Team, we developed some guidelines. We hope they will be helpful to any existing Open Source projects that would like to be associated with JEDI. As always, feedback from the field would be very much appreciated.

Three categories of projects are associated with Project JEDI:
  • Open Source project (listed on JEDI Portal)
  • Member of JEDI Alliance
  • JEDI Project (e.g. JVCL, JCL, Darth, JEDI-VCS)

In more detail:

OpenSource project listed on JEDI Portal

We provide a link on our Website to OpenSource projects that we believe provide a benefit to Delphi community. In return we request that our “Project JEDI Now” logo

is present on the project’s Website. The advantage is that both sides provide visibility to each other.

The project is totally independent from JEDI.

Requirements

  • OpenSource under at least MPL (in addition to other licenses, if desired)

How to get listed?
Please contact our JEDI Portal administrator

JEDI ALLIANCE MEMBER

Requirements

  • OpenSource under at least MPL (can have additional licenses)
  • Established, vital, and active community
  • Following Quality standards that are compatible with JEDI
  • Close cooperation on cross-releases and code sharing, if there is a need for it.
  • Clear liaison to JEDI

JEDI provides
  • Membership in our JEDI Alliance, and listing on our Website
  • Coordination of releases with JEDI, if needed
  • Technical advice on setting up a project infrastructure and on code quality issues
  • Access to JEDI community of Delphi developers
  • "JEDI Alliance Member” logo on their website with link to JEDI’s Website http://delphi-jedi.org

  • In special cases, the project will be permitted to have “JEDI” in its name.

The current project owners would be still running their projects, using their own websites, newsgroups, mailing lists, etc.

How to get associated?
If you would like to become a member of JEDI Alliance, please contact the Project JEDI Director with details of your project.

JEDI Project

Similar to JCL or JVCL

Requirements

  • As JEDI Alliance Member, plus:
  • Decisions related to the project are made by the project coordinator in coordination with the JEDI Steering Team
  • Follows JEDI "quality standards", i.e. coding style, availability of Help, etc
  • Carries “JEDI” in its name
JEDI provides/helps with
  • Mailing lists
  • Newsgroups
  • FTP server
  • Websites
  • Issue Tracker integration (our Bug Database)
  • Tools (e.g. Doc-O-Matic for Help writing)
  • Full integration into JEDI activities and planning
  • The project coordinator may become a member of the JEDI Steering Team

How to get associated?
If you would like to become a “JEDI Project”, please contact the Project JEDI Director with details of your project.
